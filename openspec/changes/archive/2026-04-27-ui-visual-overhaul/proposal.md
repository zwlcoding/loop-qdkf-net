# UI 与视觉全面升级

## 背景
游戏核心玩法已稳定（422 测试通过），但视觉层面仍是原型状态：
- 角色仅为文本标签，无精灵图
- 地形为纯色块 + emoji，无质感
- 地图为平面 2D 网格，无深度感
- UI 组件风格不统一，缺乏设计系统

## 目标
将游戏从"可玩原型"提升到"接近发行品质"的视觉水平，保持 FFTA 风格的伪 2.5D 等距视角。

## 范围

### 1. 设计系统（Theme）
- 统一色彩常量（背景/文字/强调色/边框）
- 统一字体规范（标题/正文/UI）
- 统一按钮组件（悬停/点击动效）
- 统一面板/卡片组件

### 2. 角色精灵图
5 个职业各需一套精灵图（正面/侧面/背面 或 单帧）：
- **Vanguard**（先锋/坦克）：重甲、大盾、厚重感
- **Skirmisher**（游击/斥候）：轻甲、双刀、敏捷感
- **Controller**（控场）：法袍、法杖、控制感
- **Support**（支援）：轻甲、医疗包/法杖、温暖感
- **Caster**（输出）：法袍、法球/法杖、神秘感

风格：像素风或 Q 版 2D 精灵，暗色调奇幻主题，与游戏整体风格统一。

### 3. 地形贴图
5 种地形各需一张 tile 贴图（64x64 或 128x128）：
- **Plain**（平原）：绿色草地，有草纹理
- **Mountain**（山地）：棕色岩石，有高度感
- **Urban**（城市）：灰色石板/建筑废墟
- **Forest**（森林）：深绿色，有树木元素
- **Water**（水域）：蓝色水面，有波纹

风格：等距视角像素风，与角色精灵图风格统一。

### 4. 2.5D 视觉处理
- 地图网格从平面矩形改为等距菱形（isometric diamond）
- 角色在地图上有投影/阴影
- 高度差通过 Y 轴偏移表现
- 地形边缘有过渡/混合效果

### 5. UI 重做
- 主菜单：品牌标题 + 背景渐变 + 动画按钮
- 任务选择：精美卡片 + 难度标签 + 悬停反馈
- 战斗 HUD：统一风格的面板 + 行动按钮
- 结算界面：奖励展示 + 统计数据

## 素材生成策略
所有素材由 mmx 生成，保持风格统一：
- 角色精灵图：像素风 Q 版，暗色调奇幻
- 地形贴图：等距视角像素风
- UI 元素：深色主题，蓝色强调色

## 实施顺序
1. 生成所有素材（mmx）
2. 创建设计系统 Theme.ts（kimi）
3. 替换角色渲染：文本 → 精灵图（kimi）
4. 替换地形渲染：纯色块 → 贴图（kimi）
5. 实现 2.5D 等距视角（kimi）
6. 重做 UI 组件（kimi）
7. 集成测试 + 视觉验收

## 验收标准
- 所有角色在地图上显示为精灵图，不同职业外观可区分
- 地形贴图正确渲染，风格统一
- 等距视角下地图有深度感
- UI 组件风格统一，有悬停/点击反馈
- 422+ 测试全过
- 构建成功
- 部署后手机端可正常运行
