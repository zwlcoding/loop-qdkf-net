# 设计系统规格

## 概述
创建统一的设计系统常量和组件，确保游戏所有界面风格一致。

## 需求

### REQ-1: 色彩常量
系统应导出完整的色彩常量，包括背景色、文字色、强调色、边框色、地形色、难度色、队伍色。

#### 场景: 色彩常量可用
- 前置条件: Theme.ts 已创建
- 当: 任何场景导入 COLORS
- 则: 可访问所有色彩常量

### REQ-2: 字体常量
系统应导出字体规范，包括标题、正文、UI、HUD 四个层级的字体大小。

#### 场景: 字体常量可用
- 前置条件: Theme.ts 已创建
- 当: 任何场景导入 FONTS
- 则: 可访问所有字体常量

### REQ-3: 按钮组件
系统应提供统一的按钮创建函数，支持：
- 圆角矩形背景
- 文字居中
- 悬停变色
- 点击回调
- 返回可交互对象

#### 场景: 创建按钮
- 前置条件: Theme.ts 已创建
- 当: 调用 createButton(scene, x, y, label, callback)
- 则: 创建带样式的按钮
- 且: 悬停时背景变亮
- 且: 点击时触发回调

### REQ-4: 面板组件
系统应提供统一的面板创建函数，支持：
- 圆角矩形背景
- 半透明填充
- 细边框描边
- 返回矩形对象

#### 场景: 创建面板
- 前置条件: Theme.ts 已创建
- 当: 调用 createPanel(scene, x, y, width, height, options)
- 则: 创建带样式的面板

### REQ-5: 动画函数
系统应提供淡入淡出动画函数：
- fadeIn(scene, target, duration)
- fadeOut(scene, target, duration)

#### 场景: 淡入动画
- 前置条件: Theme.ts 已创建
- 当: 调用 fadeIn(scene, target, 300)
- 则: target 在 300ms 内从透明变为不透明

#### 场景: 淡出动画
- 前置条件: Theme.ts 已创建
- 当: 调用 fadeOut(scene, target, 300)
- 则: target 在 300ms 内从不透明变为透明
