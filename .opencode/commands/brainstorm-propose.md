---
description: Brainstorm with Superpowers before creating an OpenSpec proposal
---

Use this when the idea is still fuzzy or you want stronger shaping before `/opsx-propose`.

Required behavior:
1. Load the OpenCode native skill `superpowers/brainstorming`.
2. Use that skill to explore the idea, constraints, risks, and 2-3 approaches.
3. Do not implement code.
4. When the scope is clear, produce:
   - recommended change name in kebab-case
   - one short proposal brief
   - explicit recommendation on whether it is ready for `/opsx-propose`
5. Ask whether to proceed into `/opsx-propose`.

If the user approves, continue in the same session by creating the OpenSpec proposal artifacts.

Input after `/brainstorm-propose` is the idea to explore.
